# 让记忆的"遗忘速度"自己学会适配 —— 通俗版

## 你为什么调参痛苦

现在整套记忆用**同一把尺子**量遗忘速度。不管这条记忆是:

- **"你叫什么名字"**(identity,基本每天都要想起)
- **"你刚才说想喝乌龙茶"**(working memory,一次性,过几分钟就没用了)

它们都按同一个半衰期(默认 360 小时)衰减。一把尺子量所有东西,必然有的量得太松、有的太紧。你只能去 scoreNode 那堆权重里手工补偿——这就是你调参痛苦的根源:你在用一个**排序权重**去硬补一个**本质属于衰减速度**的问题,补了东墙塌西墙。

## 我想做什么(一句话)

给每条记忆单独记一个"**稳定性**"数值,让它**自己根据被想起的频率调整遗忘速度**:常被想起的,越忘越慢;长期不被想起的,慢慢回到普通速度。你不再需要为"identity 该不该比 working memory 衰减慢"去手调——它会自己变成这样。

## 类比

想象每条记忆是一盆花,默认**浇水周期**是固定的(= 现在的 tau)。

- 一盆花你**经常打理**(= 经常被召回),我就把它的浇水周期**拉长**——它已经扎根了,扛得住更久不浇水。
- 一盆花你**好久没碰**(= 长期没被想起),我就把浇水周期**慢慢收回默认**——别让一盆早枯了的花还占着"长寿"待遇。
- 新来的花,默认周期,跟今天完全一样。

`stability` 就是"浇水周期倍率"。1.0 = 跟今天一模一样;8.0 = 衰减慢 8 倍(很不容易忘)。

## 具体怎么自动调(三个动作)

1. **被想起一次,稳定性涨一点。** 每次一条记忆被成功召回并注入对话,它就"扎根"一点,stability 上调。深召回(你明确说"还记得吗")比普通召回涨得多一点。涨有上限(8 倍),不会变成"永远不忘"。

2. **没人理,稳定性慢慢回落。** 每个清理周期,那些**这个周期内一次都没被想起**的记忆,稳定性往 1.0 退一点点(默认退 5%)。常被想起的不退。于是"曾经重要、后来不再提"的记忆会渐渐失去优待,最终按正常速度衰减、被归档——这就是"用进废退"。

3. **遗忘公式里把 stability 乘进去。** 原来是 `半衰期 = 基础值 × 情绪权重`,改成 `半衰期 = 基础值 × 情绪权重 × stability`。就这一处,所有用到衰减的地方自动跟着变。

## 为什么是"增长"而不是"拟合"

你可能问:既然是自适应,为什么不直接**算**出每条记忆的最佳半衰期?

因为算(拟合)需要知道"这条记忆**该不该**被想起、用户**有没有**真的想起来"这种**对错反馈**——现在系统里没有这个信号。而且现在的衰减是"读时现算再落库",拿落库后的曲线反推参数是**自己证自己**,数学上无效。

所以当前能做、且有研究支撑(FSRS 间隔重复、MEMORIZE)的最小可行方案是:**用"被召回"这个现成事件驱动稳定性增长**。真正的"拟合"要等以后再加一个"用户到底用没用这条记忆"的反馈信号(那是 B 档,本轮不做)。

## 对你意味着什么

- **少调一类参数**:不同 kind 的"该衰减多快"这件事,从手调变成自动。identity 自然慢、working memory 自然快。
- **默认零变化**:`stability` 新列默认 1.0,公式乘 1.0 = 跟今天逐位相同。升级当天行为不变,之后才慢慢分化。
- **4 个新旋钮,全走环境变量**(不在 UI/配置里,跟上次 #3 一样):增长强度、上下限、回落速度。不想动就用默认;觉得"长寿"上限太大就调小 `MEMORY_STABILITY_MAX`。
- **scoreNode 那 17 个排序权重本轮不动**——它们仍手调。这次只解决"衰减速度"那一半的痛苦;排序权重的自动调要等反馈信号。

## 一个需要你点头的副作用

按你之前的选择,我打算让**模型召回**(AI 主动想起来的那些)注入的记忆**也**参与"扎根"(加分 + 稳定性增长)。

这会**回退**一点你上次 #2 的决定——那时你让模型召回"只观测、不加分"。原因:稳定性需要"被成功召回"这个信号,如果模型召回不算数,稳定性就只能从本地召回学,信号会窄一半。

**只动强度/稳定性/时间,完全不碰"融合"——不会把模型的 hint 偷偷塞给本地召回。** 如果你不想回退这点,告诉我,我改成"只本地召回长稳定性,模型召回维持不加分"(信号窄一些,但 #2 完全不动)。
